K2 Moving Bodies FAQ



Q: What is K2 Moving Bodies?

K2 Moving Bodies is a project to generate light curves for the solar system objects observed with Kepler/K2. The first release will contain the light curves from objects proposed by Guest Observers in Campaigns 0-13. Hopefully there will be further releases of more objects soon. The light curves and other products from K2 Moving Bodies will shortly be available at the MAST archive as a High Level Science Product.

Q: What is asteriks?

asteriks is a code to produce light curves of asteroids observed during the Kepler/K2 mission. These moving objects require a unique data extraction that can not be performed with the normal Kepler pipeline. You can read more about asteriks and how it works in our recent paper.

Q: How do I use asteriks?

There are two ways. You can use our pregenerated products on this website or hosted at MAST. The search bar above will let you find any of the asteroids and comets currently available in our public release. Alternately, you can download our code from github here and run it for yourself. If you are having trouble with asteriks or would like to ask about a particular object, you can get in touch with the Kepler/K2 Guest Observer office directly.

Q: Why isn't my object listed?

The current version of asteriks only covers solar system objects in the first 13 campaigns of K2 data. Anything beyond these campaigns will be released in a later version of asteriks. Additionally, some objects cannot be run with asteriks. Short Cadence observations cannot be processed with asteriks, and observations of or near saturated objects can not be processed. Because of this Uranus, Neptune and all their moons are not available through asteriks. Finally some comets are too large to automatically assign an aperture too, and so are not available thorugh asteriks.

Q: How do I cite asteriks data products in my work?

There is a citation tor each object available thorugh asteriks. Click on the button at the bottom of each objects page to find the citations for asteriks and the Guest Observer proposals for the object. You will also find an acknowledgement which you can add to acknowledge the Kepler/K2 mission.

Q: What data products are available?

We provide light curves for each object in the form of a single .fits file. This file contains multiple extensions. The first extension is the 'optimal' aperture for the object. The following extensions have increasing aperture sizes. Users wanting to investigate the objects carefully should consider varying the aperture size. We also deliver an .mp4 file of each solar system object moving through the K2 observation. These are available on each objects page.

Q: How do I use asteriks .fits files?

.fits files are a common astronomy format that can be read in Python using the astropy.io.fits package, or opened with TOPCAT.

There are several extensions in the fits file, one per aperture. Each file also contains quality flags for the light curve. Bad quality flags should usually be masked. You can read more about the data format of the fits files in our readme.
